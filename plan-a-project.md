# How to Plan a Project

## Step 1: Define the Project

**Four Questions to Ask**
1. What is the project? *How is it built? What problem does it solve? Who is the desired audience?*
2. What is the Minimal Viable Product? *Think in terms of minimum features required before it can be released.*
3. What are the nice to haves?
4. When will the project be complete?

You want this to be confined to a single page. Anyone who reads it will know what the project will do.

## Step 2: Set up a workflow

**Components Required**
Project Management - Trello or something similar to set up a Kanban board. It will have the boards "TODO, DOING, DONE, BUGS / NOT SURE HOW TO DO". Add a label to signify a MVP feature and a "sprinkles" feature. 
Git - So you can manage the lifecycle of your project.

*"It is always very tempting to want to make something look fantastic and to add awesome features that will really impress a potential employer. But, if your application doesn’t perform the basic functionality it is supposed to then those awesome features aren’t going to shine. The sprinkles have no doughnut to go on."*

**Use Checklists to Break it Down Further**
For a card you're working on, add a checklist to it that defines the steps to take to break that task down further.
