# Introduction

API - Application programming interface; it gives instruction for how to interact with your application's data.

REST - An API design that combine DB resources, route paths, and HTTP verbs to allow applications to describe what action they are trying to perform. *It is the most popular API design pattern.*

## Express.js
This is the standard API framework for Node.js

* It handles all the tedious tasks such as managing sockets, route matching, error handling, etc.
* Open source.
* Huge community and support that has anything to do with APIs in Node.js

## MongoDB

The go-to non-relational DB, works well with Node.

* It's easy to get started with and scales well.
* Open source and backed by a big company.
* Tons of hosting solutions.
* ORM / ODM and other libs are some of the best for any DB.
